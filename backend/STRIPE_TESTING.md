# Stripe Test Mode Setup Guide

This document explains how to set up and test Stripe payments locally for Lumiqe.

## Prerequisites

- A Stripe account (sign up at https://stripe.com)
- Stripe CLI installed (`brew install stripe/stripe-cli/stripe` or download from https://stripe.com/docs/stripe-cli)
- The Lumiqe backend running locally on port 8000

## 1. Configure Environment Variables

Add the following to your `.env` file (never commit this file):

```
STRIPE_SECRET_KEY=sk_test_...          # From Stripe Dashboard > Developers > API keys
STRIPE_WEBHOOK_SECRET=whsec_...        # Generated by `stripe listen` (see step 3)
STRIPE_MONTHLY_PRICE_ID=price_...      # From Stripe Dashboard > Products
STRIPE_ANNUAL_PRICE_ID=price_...       # From Stripe Dashboard > Products
```

All keys must be **test mode** keys (prefixed with `sk_test_` / `pk_test_`).

## 2. Create Test Products in Stripe Dashboard

1. Go to https://dashboard.stripe.com/test/products
2. Click **Add product**
3. Create "Lumiqe Premium Monthly" with a recurring price (e.g., $9.99/month)
4. Create "Lumiqe Premium Annual" with a recurring price (e.g., $79.99/year)
5. Copy the `price_...` IDs into your `.env` as `STRIPE_MONTHLY_PRICE_ID` and `STRIPE_ANNUAL_PRICE_ID`

## 3. Set Up Webhook Forwarding

In a separate terminal, run:

```bash
stripe listen --forward-to localhost:8000/api/stripe/webhook
```

This will output a webhook signing secret (`whsec_...`). Copy it into your `.env` as `STRIPE_WEBHOOK_SECRET`, then restart the backend.

## 4. Test Card Numbers

Stripe provides test card numbers that simulate various scenarios:

| Card Number          | Scenario                        |
|----------------------|---------------------------------|
| 4242 4242 4242 4242  | Successful payment              |
| 4000 0000 0000 3220  | 3D Secure authentication        |
| 4000 0000 0000 9995  | Declined (insufficient funds)   |
| 4000 0000 0000 0069  | Declined (expired card)         |

For all test cards:
- Use any future expiration date (e.g., 12/34)
- Use any 3-digit CVC (e.g., 123)
- Use any billing ZIP code (e.g., 10001)

## 5. End-to-End Testing Flow

### Step 1: Create a test customer

Register a new user account through the Lumiqe frontend or API:

```bash
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "test@example.com", "password": "TestPass123!", "name": "Test User"}'
```

### Step 2: Initiate checkout

Log in and create a Stripe Checkout session:

```bash
curl -X POST http://localhost:8000/api/stripe/create-checkout-session \
  -H "Authorization: Bearer <your_jwt_token>" \
  -H "Content-Type: application/json" \
  -d '{"price_id": "<STRIPE_MONTHLY_PRICE_ID>"}'
```

Open the returned `checkout_url` in a browser and complete payment with test card `4242 4242 4242 4242`.

### Step 3: Verify webhook delivery

Check the terminal running `stripe listen`. You should see events like:

```
checkout.session.completed
customer.subscription.created
invoice.payment_succeeded
```

### Step 4: Confirm premium status

Verify the user now has premium access:

```bash
curl http://localhost:8000/api/auth/me \
  -H "Authorization: Bearer <your_jwt_token>"
```

The response should show `"is_premium": true` and include the `stripe_subscription_id`.

## 6. Testing Subscription Cancellation

```bash
curl -X POST http://localhost:8000/api/stripe/cancel-subscription \
  -H "Authorization: Bearer <your_jwt_token>"
```

Check `stripe listen` output for `customer.subscription.deleted` event, then verify `is_premium` is `false`.

## 7. Useful Stripe CLI Commands

```bash
stripe trigger checkout.session.completed    # Manually trigger a webhook event
stripe logs tail                              # Stream API request logs
stripe events list --limit 10                 # List recent events
```
